Getting Started

Your First Simulation

  1. Click "Explore Sims" from the main menu
  2. Choose a simulation (Three-Body Problem recommended for beginners)
  3. Wait for the 3D visualization to load
  4. Click the Start button (‚ñ∂) to begin

You'll see three colored spheres (red, green, blue) moving and leaving trails. They're pulling on each other with gravity, just like real celestial bodies! The default Figure-8 orbit shows three bodies chasing each other in a beautiful repeating pattern.

First-Timer Tip: Let the Figure-8 run for 30+ seconds to see the complete pattern. It's mesmerizing!

Understanding the Interface

Header Information

Time Display: Shows simulation time (not real-world time). If you set speed to 2x, simulation time moves twice as fast.

Energy Display: Critical accuracy indicator! Energy should stay constant in a closed system.

  • Good: Drift less than 0.01%
  • Acceptable: 0.01% - 0.1% drift
  • Problem: Over 1% drift - reduce time step!

FPS (Frames Per Second): Rendering smoothness. 60 FPS is ideal, 30-60 is good, below 30 means reduce trail length.

Camera Controls

Desktop/Laptop:

  • Left Click + Drag: Rotate view around the scene
  • Right Click + Drag: Pan the view
  • Scroll Wheel: Zoom in/out

Tablets/Phones:

  • One Finger: Rotate view
  • Pinch: Zoom
  • Two Fingers: Pan

Pro Tip: Try different viewing angles! The Figure-8 looks completely different from above versus from the side.

Control Panel Guide

Simulation Presets

Figure-8 Orbit: Three equal-mass bodies in a periodic figure-8 pattern. Discovered in 1993, this elegant solution repeats every 6.32 time units. Perfect for showing that three-body systems CAN be predictable!

Lagrange Triangle: Bodies maintain an equilateral triangle while orbiting their center of mass. Perfectly stable for equal masses. Example: Jupiter's Trojan asteroids!

Chaotic System: Unpredictable motion demonstrating chaos theory. Expect close encounters and wild swings. Sensitive to initial conditions - run it twice, get different results!

Custom: Your experimental playground! Set your own masses, positions, and velocities. Try discovering new periodic orbits!

Playback Controls

  • Start (‚ñ∂): Begin or resume simulation
  • Pause (‚è∏): Freeze time to examine configurations
  • Reset (‚Üª): Return to initial conditions

Key Parameters

Simulation Speed (0.1x - 5.0x): Controls how fast simulation runs. Higher = faster but may reduce accuracy. Default 1.0x is recommended.

Trail Length (100 - 5000 points): How long orbital trails persist. Long trails show beautiful patterns but may impact performance. Default 2000 is balanced.

Time Step dt (0.0001 - 0.01): Accuracy vs speed tradeoff. Smaller = more accurate but slower. Larger = faster but less accurate. Default 0.001 works well for most cases.

Important: If you see high energy drift, DECREASE the time step for better accuracy!

Tips & Best Practices

Getting the Best Results

  • Start Simple: Begin with Figure-8 or Lagrange presets before trying chaotic systems
  • Watch Energy: Keep an eye on energy drift percentage. It's your accuracy gauge!
  • Adjust for Performance: If simulation runs slow, reduce trail length first, then increase time step
  • Experiment with Views: Different camera angles reveal different aspects of the motion
  • Use Pause: Freeze time to examine interesting configurations or take screenshots

Understanding Behavior

Periodic Orbits (Figure-8, Lagrange): These repeat perfectly. Energy should stay rock-solid constant. If it drifts, something's wrong with your settings.

Chaotic Systems: Never repeat exactly. Small changes lead to completely different outcomes. This is the famous "butterfly effect" - the hallmark of chaos theory!

Close Encounters: When bodies pass near each other, they experience strong forces. This requires smaller time steps for accuracy. If energy spikes during close encounters, reduce dt.

Troubleshooting

Common Issues & Solutions

Simulation Running Slowly:

  • Reduce trail length to 1000 or lower
  • Increase time step slightly (but watch energy!)
  • Disable velocity vectors if shown
  • Close other browser tabs

High Energy Drift:

  • Decrease time step (dt) - this is the main solution
  • For chaotic systems, try dt = 0.0005 or smaller
  • Check for close encounters between bodies
  • Consider using a different preset

Bodies Moving Too Fast/Slow:

  • Adjust simulation speed multiplier
  • Check initial velocity values in custom presets
  • Reset and try a different preset

Trails Look Choppy:

  • Increase trail length for smoother appearance
  • Reduce simulation speed for better visualization
  • Check that FPS is above 30

Bodies Escape or Collide:

  • This can happen in chaotic systems - it's actually realistic!
  • Try a smaller time step for better accuracy near close encounters
  • Reset and try again - chaotic systems behave differently each run
  • Use stable presets (Figure-8, Lagrange) if you want predictable behavior

Remember: Chaos means unpredictability is normal! If the chaotic preset does something wild, that's physics, not a bug.
